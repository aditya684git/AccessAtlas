# Quick Fix: Add Test Tags to See Model & OSM Data

## The Problem
You're only seeing user-added tags because:
1. **Model tags** need to be generated by your trained model
2. **OSM tags** may not exist in your current map area

## Instant Solution: Add Test Data

### Step 1: Open Browser Console
Press `F12` or `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac)

### Step 2: Run This Command
```javascript
await window.addTestTags()
```

**Note:** Make sure to include `await` - the function now fetches real addresses from Nominatim API!

### Step 3: Refresh the Page
Press `F5` or `Ctrl+R`

### What This Does
- Adds 5 test model tags (with confidence scores)
- Adds 5 test OSM tags
- All around your current map location

### Step 4: Open Debug Panel
Click the **ğŸ› Debug** button to verify:
- You should now see Model: 5
- You should now see OSM: 5

## Real OSM Data

To get REAL OSM accessibility features:

### Option 1: Move to a Different Location
Many areas don't have OSM accessibility data. Try:
- University campuses
- City centers
- Public transit stations
- Government buildings

### Option 2: Check OSM Logs
Open console and look for:
```

ğŸ—ºï¸ OSM API returned X elements
```

If it returns 0 elements, that area has no OSM accessibility data.

## Real Model Tags

To get model-generated tags:

### Your model needs to:
1. Make predictions on map tiles
2. Create Tag objects with:
   ```typescript
   {
     id: 'unique-id',
     type: 'Ramp' | 'Elevator' | etc,
     lat: number,
     lon: number,
     source: 'model',  // â† CRITICAL!
     confidence: 0.85,  // 0-1 score
     timestamp: new Date().toISOString(),
   }
   ```
3. Save them to localStorage or backend

### Check if model is running:
```javascript
// In console:
const tags = JSON.parse(localStorage.getItem('accessibility_tags') || '[]');
console.log('Model tags:', tags.filter(t => t.source === 'model').length);
```

## Remove Test Data

When done testing:
```javascript
window.clearTestTags()
```

Then refresh the page.

## Troubleshooting

### "window.addTestTags is not a function"
- Make sure the page has fully loaded
- Check that the dev server is running
- Try refreshing and running again

### Test tags appear but no colors
- Open Debug Panel (ğŸ› button)
- Check that all source toggles are ON
- Set confidence slider to 0%
